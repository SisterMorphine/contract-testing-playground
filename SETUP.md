# Contract Testing POC with Jest and Pact

This is a working proof of concept (POC) for contract testing using Jest and Pact.

## Project Structure

```
├── consumer/
│   └── tests/
│       └── consumer.test.ts       # Consumer (client) test - defines expectations
├── provider/
│   ├── userService.ts             # Provider (server) implementation
│   └── tests/
│       └── provider.test.ts        # Provider test - verifies contract compliance
├── pacts/
│   └── UserApp-UserService.json    # Generated contract file
├── jest.config.ts                 # Jest configuration for TypeScript
├── tsconfig.json                  # TypeScript configuration
└── package.json
```

## How It Works

### 1. Consumer Test (consumer/tests/consumer.test.ts)
- Defines what the consumer (UserApp) expects from the provider
- Uses `PactV3` to create a mock server
- Defines interactions (requests/responses)
- Generates a contract file in `pacts/` folder

### 2. Provider Test (provider/tests/provider.test.ts)
- Starts the actual provider service (UserService)
- Uses `Verifier` to check if the provider satisfies the contract
- Ensures the provider implementation matches consumer expectations

### 3. Contract File (pacts/UserApp-UserService.json)
- Generated by the consumer test
- Contains all defined interactions
- Used by provider test for verification

## Setup

```bash
# Install dependencies
npm install

# Run all tests (both consumer and provider)
npm test

# Run only consumer test (generates contract)
npm run test:consumer

# Run only provider test (verifies contract)
npm run test:provider
```

## Test Flow

1. **Consumer Test** → Generates contract (`pacts/UserApp-UserService.json`)
2. **Provider Test** → Verifies provider satisfies the contract

## Key Configurations

### package.json
- No `"type": "module"` needed - ts-jest handles ES modules automatically
- Simple test scripts (no experimental flags)

### jest.config.ts
- `preset: 'ts-jest'` - Handles TypeScript compilation
- `testEnvironment: 'node'` - Run in Node.js environment
- `extensionsToTreatAsEsm: ['.ts']` - Proper ES module support

### tsconfig.json
- `target: ES2020` - Modern JavaScript target
- `module: ES2020` - ES modules
- `strict: true` - Full type checking enabled

## Adding New Interactions

To add a new contract interaction:

1. **Add to Consumer Test** (`consumer/tests/consumer.test.ts`):
   ```typescript
   test('POST /users', async () => {
     await pact.addInteraction({
       uponReceiving: 'a request to create a user',
       withRequest: {
         method: 'POST',
         path: '/users',
         body: { name: 'Jane', status: 'ACTIVE' }
       },
       willRespondWith: {
         status: 201,
         body: { id: 3, name: 'Jane', status: 'ACTIVE' }
       },
     });

     return pact.executeTest(async (mockServer) => {
       const response = await axios.post(`${mockServer.url}/users`, {
         name: 'Jane',
         status: 'ACTIVE'
       });
       expect(response.status).toBe(201);
     });
   });
   ```

2. **Implement in Provider** (`provider/userService.ts`):
   - Add the corresponding endpoint

3. **Run Tests**:
   ```bash
   npm test
   ```
